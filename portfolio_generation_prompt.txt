ISSUE 1 — COLOUR SYSTEM: FOLLOW EXACTLY, DO NOT DEVIATE
═══════════════════════════════════════════════════════════════

Palette definitions:
  PRIMARY:    #0FBA4C  (green)
              #2168EB  (blue)
  SECONDARY:  #800FF2  (purple)
  ACCENT:     #0B0859  (deep navy)
              #1E0D8C  (dark indigo)
  BG:         #FFFFFF  (white — default page background)
  TEXT:       #000000  (black — default body text)

TEXT COLOUR RULES — THIS IS THE MOST IMPORTANT SECTION

Rule 1 — Default text:
  When the background of a section or element is #FFFFFF (white),
  all text must be #000000 (black).

Rule 2 — Light-on-dark override:
  When the background of a section, card, button, banner, navbar,
  footer, tag, badge, or any other element is filled with ANY of
  the following colours:
    #0FBA4C  (primary green)
    #2168EB  (primary blue)
    #800FF2  (secondary purple)
    #0B0859  (accent deep navy)
    #1E0D8C  (accent dark indigo)
  Then ALL text, headings, labels, icons, and links inside that
  element must be #FFFFFF (white). Never use #000000 on a dark
  or saturated background.

Rule 3 — Never mix:
  Do not place #000000 text on any PRIMARY, SECONDARY, or ACCENT
  background. Do not place coloured text (e.g. #0FBA4C or #2168EB)
  on a #000000 background unless explicitly instructed.

COLOUR USAGE RULES

PRIMARY #0FBA4C (green):
  - Main CTA buttons (e.g. "See My Work", "Send Message")
  - Active nav link indicator
  - Hover state highlights on links and buttons
  - Text inside must always be #FFFFFF when used as fill/background

PRIMARY #2168EB (blue):
  - Hyperlinks in body text
  - Section dividers or accent lines
  - Outlined secondary buttons: border #2168EB, text #2168EB on
    white background. If button is filled with #2168EB, text
    must switch to #FFFFFF.

SECONDARY #800FF2 (purple):
  - Skill tags and badge backgrounds
  - Highlighted labels
  - Text inside must always be #FFFFFF

ACCENT #0B0859 / #1E0D8C (deep navy / dark indigo):
  - Hero section background
  - Footer background
  - Global header background
  - Dark-themed panels
  - All text, headings, and subtext inside must be #FFFFFF

BG #FFFFFF:
  - Default page background
  - Card backgrounds inside dark sections
  - Text on #FFFFFF is always #000000

TEXT #000000:
  - Default body copy, paragraphs, labels
  - Only used when background is #FFFFFF
  - Never use on PRIMARY, SECONDARY, or ACCENT backgrounds

CONTRAST REMINDER:
Any element with a background from PRIMARY, SECONDARY, or ACCENT
is a dark or saturated surface. All content on it (text, subtext,
icons, captions) must be #FFFFFF. Apply text colour explicitly to
every element based on its own background, not the parent page
background. Do not infer — be explicit on every element.

Define all colours as CSS custom properties in :root in
styles/styles.css. Never hardcode colour hex values in HTML.
